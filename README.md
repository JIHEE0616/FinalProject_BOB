제가 맡은 프로젝트의 주요 기능은 다음과 같습니다.

 ## 개발 내용
 - **[ 메인페이지 - 알림창 ]**
1) 로그인 시 '종'모양의 알림창 아이콘이 뜸
2) 아이콘 클릭시 장인신청진행상황여부에 따른 내용이 출력 -> 심사중,승인내용,거절내용,박탈내용
3) 관리자는 매일 꼭 해야할 내용 출력 -> 문의답변,환불승인

- **[ 챗봇 - 군침이 ]**
1) 카카오챗봇API 사용
2) 헤더오른편에 위치한 챗봇추가 버튼 클릭 시 '군침이'를 자동으로 추가할 수 있음
3) 군침이 추가 후 챗봇대화를 통해 사용자의 문의에 챗봇이 자동으로 대답해줌
4) 사용자가 입력할 것 같은 발화들을 예상하여 알맞은 대답으로 설정해둠
5) 상담직원 연결시 상담직원이 대답할 수 있게 설정

- **[ 관리자 페이지 ]**
1) 메인

-> Javascript를 통해 회원수, 매출액, 판매물품수, 직원수가 동적으로 올라가게 함

-> Chart.js를 활용하여 판매물품 카테고리별 세분화 그래프, 올해 목표 매출액과 현재 매출액 그래프를 시각적으로 표현


2) 회원관리

-> 전체 회원리스트를 가입일기준 오름차순으로 출력

-> 이름, 아이디, 닉네임으로 검색 가능

-> 기본 페이징처리 + 검색 후 페이징 처리 가능

-> 회원 클릭시 회원 세부정보 확인 가능하며 탈퇴 가능


3) 클래스 장인 관리

-> 장인 신청한 회원 전체 출력

-> 심사필요 우선출력, 나머지는 심사일 기준 내림차순으로 출력

-> 아이디, 강사명으로 검색 가능

-> 기본 페이징처리 + 검색 후 페이징 처리 가능

-> 장인이 된 회원 클릭 시 해당 회원의 장인정보 및 클래스 정보 확인 가능

-> 장인 자격 발탈 가능

-> 박탈,거절 클릭 시 사유 확인 가능


4) 1:1 문의 관리

-> CUSTOMER CENTER 1:1문의에 적힌 문의 중 답변 하지 않은 글을 문의 날짜 기준으로 오름차순으로 출력

-> 문의글을 클릭하여 답변 입력 가능

-> 페이징처리

5) 원데이클래스/마켓 문의 관리

-> 원데이클래스/마켓에 적힌 문의글 중 답변 하지 않은 글 전체 출력

-> 원데이클래스와 마켓 분류해서 확인 가능

-> 글 클릭시 해당 글이 있는 위치로 이동

-> 페이징처리


6) 환불 관리

-> 회원이 신청한 환불내역이 날짜기준으로 오름차순으로 출력

-> 환불승인 버튼 클릭시 회원이 환불 신청시 입력했던 환불사유를 확인 할 수 있으며, 아임포트API를 통해 환불처리

-> 페이징처리

7) 상품 관리

-> 마켓에 등록된 모든 상품 출력, 재고량 기준으로 오름차순으로 출력

-> 제품명, 재고량으로 검색 가능

-> 상품이미지 클릭 시 해당 상품 위치로 이동

-> 상품정보수정 버튼 클릭시 해당 상품 정보 수정 페이지로 이동

-> 기본 페이징처리 + 검색 후 페이징 처리 가능

### 2. 후기  

 세미 프로젝트가 끝나고 얼마 되지 않아 바로 시작한 파이널 프로젝트였습니다. 
그래서그런지 주제를 선정하는 과정이나 ERD, 화면레이아웃은 처음보단 다들 어느정도는 익숙해했던게 사실입니다. 하지만 스프링에 대해 완벽히 이해하지 못한 상태에서 시작한 파이널이였어서 스프링을 쓰는 것 자체가 너무 큰 부담이었습니다. 검색도 많이 하고 이리저리 해보다보니 또 지금은 끝이났다는게 믿기지않고 뿌듯하기도 합니다. 

챗봇을 통해 API도 사용해 볼 수 있었고 새로운 것을 해 볼 수 있어 재미있었습니다. 또 관리자페이지를 담당하여 전보다 더 다양한 Sql문을 사용해 볼 수 있어 좋았고, 부족하다고 느꼈던 부분을 조금이나마 보완할 수 있었던 시간이었던 것 같습니다. 

다양한 코드를 접하고 또 나와 다른 풀이방법을 보면서 앞으로 더 나아갈 수 있었으며, 세미때와 마찬가지로 모든 문제를 스스로 해결해나가며 완성시켰던 것 같습니다.
이러한 프로젝트 경험이 추후 많은 도움이 되리라 자부합니다.

감사합니다.
